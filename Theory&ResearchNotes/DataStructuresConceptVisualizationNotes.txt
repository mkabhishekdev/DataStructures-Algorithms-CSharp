    * Reverse a Linked List
      ----------------------
      1. Visualize movements step by step.
      2. Think of it has you are reversing the links to the addresses stored in next pointer.
      3. What you need? You will start with currentNode = head (it points to head)
                        - Than prev = null (because prev pointer to first node points to null)
      4. Think here? you will loose the connection to next pointer since you are not saving it anywhere,
        so create next = curr.next (this will keep the pointer to the next link safe before you reverse)
      5. Now think that with the below two lined you reversed the first node,
             next = curr.next;
             curr.next = prev;
      6. Think that now you moved to the second node(visualize), so whatever was your current node in the previous
         step will become previous now:
         prev = curr;
      7. So now the current pointer, where will you get the value from? remember the very first step where you kept
         the value saved in 'next' pointer.
         curr = next;
      8. You will write all these within a while(current != null). Also where will you find the updated head value
         from? From the 'prev'
      9. So writing all the above logic as a code snippet:
         
         Imagine: When you think of reversing, just think of the 'very first node'
                  For the very first node:
                   step 1. what is the prev i.e prev = null
                   step 2. before even reversing, you need to definitely save the next address value or else this will 
                        be lost after reversing.
                        So you do what? i.e next = curr.next;
                   step 3. In order to come back to the loop, you need to get to the next element
                           where have you kept it?: its in next [step2 remember] i.e curr = next;
                   step 4. you reverse i.e curr.next = prev;
                   step 5. once you have done the reversal, prev can't be obviously null
                           So what it will be?: it has to be the current element which just did a reversal
                           i.e prev = curr;
                   step 6. you need to now point the updated head, the updated head would be the last element before 
                           current became null.
                           So where is that, in the 'prev' i.e prev = curr [step 2], update: head = prev;

         Node prev = null; [step 1]
         Node curr = head;
         Node next;

         while(curr != null)
         {
             next = curr.next; [step 2]
             curr.next = prev;  [step 4]
             prev = curr;   [step 4]
             curr = next;  [step 3]
         }
         head = prev; [step 6]

* Drawback of linear queue can be handled using Circular queue.
  What is the drawback?
    - In linear queue, even though if unused memory is present. If the linear queue as reached its maximum size - it
      would not allow to insert. This problem could be solved using a circular queue.

* When thinking of implementing a circular queue always think that in a usual queue: rear.next -> null at last but in
  circular queue the: rear.next -> front i.e will point to the first node, this way circular motion is adopted.

5. DATA-STRUCTURE: TREE
*****************************************
    - Parent node = Immediate predecessor of a given node
    - Child node= Immediate successor of a given node
    - Leaf node = Nodes which do not have any child node
    - Non Leaf node(internal nodes) = Nodes which have atleast one child
    - Path = Sequence of consecutive edges from source node to destination node 
    - Ancestor = any predecessor node on the path from the root to that node
    - Descendant = any successor node on the path from that node to the leaf node
    - Sibling = Children of the same Parent
    - Degree = Degree of node is number of children of that node (the degree of the 'leaf node' is always 0)
          * Maximum degree among all nodes
    - Depth of a node = Length of Path from "root" to "that node"(the length to the specific node is what differs 'Depth' of a node from 'Height' of a node)
    - Height of a node = Number of edges in the longest path from that "node" to the "leaf node"
    - Level of a node =  number of edges from the "root" to the "given node" (Depth = Level)
    - 'n' nodes = '(n-1)' edges

    * Binary-Tree
    -------------
        - In a binary tree, each node can have atmost 2 childrens (i.e they can have 0, 1, 2 but not more than 2 childrens)
        - Types of Binary tree = Full/Proper/Strict binary tree, Complete binary tree, Perfect binary tree, Degenerate binary tree
    
    DFS(Depth First Search) below:
    - InOrder traversal = left -> root -> right
    - PreOrder traversal = root -> left -> right
    - PostOrder traversal = left -> right -> root

    BFS(Breadth First Search): go through each row in the tree like 'level'(say level 0,level1, level2, level3) & write
                               down the modes
