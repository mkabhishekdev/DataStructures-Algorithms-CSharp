                        be lost after reversing.
                        So you do what? i.e next = curr.next;

         Node curr = head;
         Node next;

             curr.next = prev;  [step 4]
             prev = curr;   [step 4]
             curr = next;  [step 3]
         }

5. DATA-STRUCTURE: TREE
*********************************************************************************************************************

    - Parent node = Immediate predecessor of a given node
    - Child node= Immediate successor of a given node
    - Leaf node = Nodes which do not have any child node
    - Non Leaf node(internal nodes) = Nodes which have atleast one child
    - Path = Sequence of consecutive edges from source node to destination node 
    - Ancestor = any predecessor node on the path from the root to that node
    - Descendant = any successor node on the path from that node to the leaf node
    - Sibling = Children of the same Parent
    - Degree = Degree of node is number of children of that node (the degree of the 'leaf node' is always 0)
          * Maximum degree among all nodes
    - Depth of a node = Length of Path from "root" to "that node"(the length to the specific node is what differs 'Depth' of a node from 'Height' of a node)
    - Height of a node = Number of edges in the longest path from that "node" to the "leaf node"
    - Level of a node =  number of edges from the "root" to the "given node" (Depth = Level)
    - 'n' nodes = '(n-1)' edges
 
    * Binary-Tree
    -------------
        - In a binary tree, each node can have atmost 2 childrens (i.e they can have 0, 1, 2 but not more than 2 childrens)
        - Types of Binary tree = Full/Proper/Strict binary tree, Complete binary tree, Perfect binary tree, Degenerate binary tree
    
    DFS(Depth First Search) below:
    - InOrder traversal = left -> root -> right
    - PreOrder traversal = root -> left -> right
    - PostOrder traversal = left -> right -> root

    - Pre → Root comes first
      In → Root in the middle
      Post → Root comes last

    BFS(Breadth First Search): go through each row in the tree like 'level'(say level 0,level1, level2, level3) & write
                               down the nodes
     
    * Lowest Common Ancestor in BST
    ----------------------------------------------------
        - Get the Max and Min of the [firstnode, secondnode], compare them with the root: where both the conditions
          fail is your LCA [check your code folder]   
    
    * Binary Search Tree(BST)
    -----------------------------------------------------
        - BST are trees in which left of a 'node' is lesser than the node and right of a 'node' is greater than the node.
        - 'Duplicate elements' are not allowed in a BST. Important point.

    * AVL Trees
    -------------------------------------------------------------
        - AVL trees are a special form of 'Binary Search Trees'. It is a 'Self BALANCING' BST. (STRICTLY BALANCED)
        - The balance factor(i.e left - right) should be either -1,0,1. If >1, the tree is unbalanced and you need to balance it. 
        - A Binary search tree is a AVL tree, if the difference of (leftside - rightside) of a 'node' is not greater than 1
        - Why we need AVL tree? In general insertion to a Binary Search tree also takes O(n) this is the same as an array or
          a linked list.
          AVL tree takes O(logn)
          
    * Why Height of a Tree(i.e balanced tree) is 'logn' ?
    -----------------------------------------------------
    First: what is “height” of a binary tree?

    Height = number of levels from root to the deepest leaf

    Example:

        Level 1:        1
        Level 2:      2   3
        Level 3:    4  5 6  7

        Height here = 3

        Key idea (no logs yet)

        In a binary tree,
        👉 each level can have at most double the nodes of the previous level

     Level	Max nodes at that level
     1	1
     2	2
     3	4
     4	8
     5	16

    This doubling is the core reason logs appear.

    Imagine building the tree step by step
      Suppose you keep adding levels:

      After 1 level → 1 node
      After 2 levels → 1 + 2 = 3 nodes
      After 3 levels → 1 + 2 + 4 = 7 nodes
      After 4 levels → 1 + 2 + 4 + 8 = 15 nodes
    
      So roughly:

      Height 1 → ~1 node
      Height 2 → ~2 nodes
      Height 3 → ~4 nodes
      Height 4 → ~8 nodes
      Height 5 → ~16 nodes


      Every time you increase height by 1, nodes roughly double

      Now reverse the thinking (this is the key mental flip)

      Instead of asking:
      “If height is H, how many nodes?”

       Ask:

      “If I have N nodes, how many levels do I need?”

      Example:

      8 nodes → need 3 levels

      16 nodes → need 4 levels

      32 nodes → need 5 levels

      Notice something?

    👉 To reach big numbers of nodes, height grows very slowly

    This “how many times do I double to reach N?” is exactly log₂(N)

    Let’s say:

    1 → 2 → 4 → 8 → 16 → 32


    Count how many steps (levels) it took:

      To reach 8 → 3 steps

      To reach 16 → 4 steps

      To reach N → log₂(N) steps

    So:

    Height ≈ log₂(N)

    Why people say “height of binary tree is log n”

    This statement is true ONLY for a balanced binary tree.

    Balanced means:

    Left and right subtrees are roughly equal

    Tree grows wide, not tall

    Balanced tree:
    Height = O(log n)


Red Black Tree
-------------------------------------------------------------------------
    
  * Rules to be a Red black tree
    - Red black tree is a 'Self BALANCING' BST. Every node is either black or red. (STRICTLY BALANCED)
    - Root is always 'Black'.
    - Every leaf node which is NIL is black. 
    - If node is 'Red' then its children are 'black'.
    - Every path from a node to any of its descendant NIL node has same number of black nodes.
  

Splay Tree
---------------------------------------------------------------------------

  - Splay tree is a 'Self ADJUSTING' BST. (ROUGHLY BALANCED)
  - In general, BST in worst case as time complexity: O(n). Trees like AVL, RedBlack make the time complexity: O(logn)[that is their major importance]
  - Splay tree is applied to reduce the time complexity further more.
  - In Splay trees, you first search for the element(using BST) + than do Splaying.
  - Splaying is a process in which each operation on a element will rearrange the tree, so that it will become 'root' of the tree.
  - After splaying process, the time complexity can be O(1) since the element being checked could be the root/node 'nearer' to the root.
  - Major application of Splay tree: Implementation of 'CACHES'.
  - Fastest type of BST for many practical application.
