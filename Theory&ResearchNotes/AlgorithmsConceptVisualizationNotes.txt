2. How to Analyze Time Complexity in a Code Snippet
****************************************************

Ask yourself these key questions:

1. How many times does each line run?
Look at each loop (for, while, foreach)

Nested loops? Multiply the iterations.

🔁 A single loop from 0 to n → O(n)
🔁 Nested loop → O(n²), O(n³), etc.

2. Are there recursive calls?
Count how many times the function recurses, and how the input shrinks.

Example:
BinarySearch(arr, left, right) → halves input each time → O(log n)
Fibonacci(n) → calls itself twice → exponential → O(2ⁿ)

3. Look at the input size (n)
Does the running time depend on input size?
If you're iterating over an array of n items, it’s probably O(n).

4. Ignore constants and lower-order terms
O(2n) → O(n)
O(n + log n) → O(n)

Just focus on the highest growing term.

🔍 Quick Examples:
✅ Example 1 – O(n):

for (int i = 0; i < n; i++)
{
    Console.WriteLine(i);
}
Runs n times → O(n)

✅ Example 2 – O(n²):

for (int i = 0; i < n; i++)
{
    for (int j = 0; j < n; j++)
    {
        Console.WriteLine(i + j);
    }
}
Runs n * n times → O(n²)

✅ Example 3 – O(log n):

while (n > 1)
{
    n = n / 2;
}
Divides n every time → O(log n)

🔑 Summary:

Code Pattern	                                 Time Complexity
Single loop over n	                               O(n)
Nested loops	                                     O(n²), O(n³), etc.
Divide by 2 each time	                            O(log n)
Loop inside loop over different inputs	             O(n * m)
Recursion doubling calls	                         O(2ⁿ)




3. How to Analyze Space Complexity (Simple Way)
************************************************************
👉 Ask:

How much extra memory is being used as input size (n) grows?

📋 Common Patterns for Space Complexity:

Code Pattern	                                  Space Complexity	                        Reason
Just a few variables (int, bool, etc.)	              O(1)	                          Constant memory
New array or list of size n	                       O(n)	                       Space grows with input
2D array (n x n)	                                   O(n²)	                    Space grows as square of input
Recursion (depth = n)	                             O(n)	                     Stack memory for each call
Divide and conquer recursion	                      O(log n)	                  Stack depth based on halves


🔥 Simple Examples:

✅ Example 1 — O(1) Space:

int sum = 0;
for (int i = 0; i < n; i++)
{
    sum += i;
}
Only sum and i used.

Memory doesn't depend on n.

Space = O(1)

✅ Example 2 — O(n) Space:

int[] newArr = new int[n];
for (int i = 0; i < n; i++)
{
    newArr[i] = i * 2;
}
newArr size grows with n.

Space = O(n)

✅ Example 3 — O(n²) Space:

int[,] matrix = new int[n, n];
2D array of size n × n.

Space = O(n²)

✅ Example 4 — O(n) Space (Recursion):

void Print(int n)
{
    if (n == 0) return;
    Console.WriteLine(n);
    Print(n - 1);
}
Each call waits for the next → depth = n.

Stack memory grows with n.

Space = O(n)

✅ Example 5 — O(log n) Space (Divide and Conquer):

void BinarySearch(int[] arr, int left, int right)
{
    if (left > right) return;
    int mid = (left + right) / 2;
    BinarySearch(arr, left, mid - 1);
    BinarySearch(arr, mid + 1, right);
}
Each call divides problem in half.

Stack depth = log n.

Space = O(log n)

🎯 Final Shortcuts

Look for...	                                                    Likely Space
Only primitive variables	                                         O(1)
New array/list proportional to input	                             O(n)
Nested structures (matrix, graph)	                            O(n²), O(n³)
Simple recursion	                                                  O(n)
Divide and conquer recursion	                                     O(log n)

- Insertion Sort
  ------------
  Main-Idea:
  -----------
  1. You need to visualize dividing the given unsorted array into two sublists: sorted sublist on the left &
     unsorted sublist on the right.
  2. Initially start the sorted sublist with just one item on the left.
  3. Pick the first item in the unsorted list and start comparing with the sorted list and keep on moving item
     from unsorted sublist to sorted sublist.


- Selection Sort
  ------------
  Main-Idea:
  -----------
  1. Select the first element as the minimum element to start with.
  2. Compare each element present in the list and find the minimum value.
  3. If minimum value found is not the same as the minimum element set initially(i.e first element to start with), swap the 
     elements.

- Quick Sort
------------
  Main-Idea:
  -----------
  1. Also known as Partition sorting, main idea is to partition the given list, than sorting each sub partitioned
     sets. 
  2. Keep variables say lowerbound,upperbound,start,end and set the first element as 'Pivot' to start with.
  3. Outer while loop will be definitely (start < end), inside check for conditions and Swap.
  4. Note, the Pivotal element will need to be swapped when you reach to the outer while condition if start > end, which
     is not true and hence swap it. Create first a Partition(Array, lowerbound, upperbound) function and than have the 
     Quicksort function which will handle the sub-partitioning of the elements.

1.8.1: Asymptotic notations
---------------------------
* O big-oh = upper bound
* Omega = lower bound
* Theta = average bound

Below is always the basic complexity order in general:
1 < logn < root(n) < n < nlogn < n2(square) < n3(cube) < ........... < 2(power of n) < 3(power of n) < ......... < n(power of n)


1.8.2: Asymptotic notations(BigO-Omega-Theta)
---------------------------------------------
* For a function, when you are not able to find the average bound(omega), upperbound(bigo) and lowerbound(theta) become useful.
  Example:
          f(n) = n!
          1*1*1*1 <= 1*2*3*...*n <= n*n*n*n2
          omega(1)                  bigo(n(power of n))
* Always average bound(theta) is preferred, if you can find than it is better.
* An analogy here, think of you want to buy a new phone - you enquire with some of your friends. Some will say you may get around Rs 20,000
  and some may say minimum you will get a good phone around Rs 10,000. Bestcase is Rs 20,000 and worst-case his Rs10,000. But upper-bound
  or lowerbound are not the exact value you got. So average bound is say around Rs 15,000 and that gives a exact price point at which you
  can buy a phone.

   are modes of representing whether they are upperbound, lowerbound or average bound.
   Example:
   1. Linear Search
          BestCase(n) = 1 -> O(1),Omega(1),Theta(1)

  2.6.1: Binary Search Iterative Method

      * int BinarySearch(int[] arr, int size, int key)
        {
            int low = 0, high = n-1;

            int mid = (low+high)/2;
          
            while(low < high)
            {

                }

  2.6.2: Binary Search Recursive Method
  --------------------------------------
    *  Algorithm RecursiveBSearch(low,high,key)
       {
           if(low == high)                                   // 1 UNIT
           {
               if(key == a[low])
               {
                   return low;
               }
           }
           else
           {
               int mid = (low+high)/2;                        // 1 UNIT
               
               if(key == a[mid])                              // 1
               {
                  return mid;
               }
               if(key < a[mid])                                    // BELOW IF-ELSE: T(N/2)
               {
                   return RecursiveBSearch(low,mid-1,key);
               }
               else
               {
                   return RecursiveBSearch(mid+1,high,key);
               }
           }  
       }

       T(N) = T(N/2) + 1, N > 1


  
  2.7: The Sorting techniques & how to talk about their Whys:
  ----------------------------------------------------------------------------------------

       1. QUICK SORT (THE BEST)
            WHY?  - The best, uses Best case = O(nlogn), Worst case = O(n2) time 
                  - No extra space
                  - Fastest on average, cache-friendly, used conceptually in many standard libraries
                  - https://www.youtube.com/watch?v=-qOVVRIZzao&list=PLDN4rrl48XKpZkf03iYFl-O29szjTrs_O&index=37
               
              * Key mental model (Algorithm): QuickSort → “Pick pivot, partition, fix pivot forever”

                 1. Pivot you start with assuming the first element, i = leftmost element & j = rightmost element
                 2. Move i right: while(A[i] <= Pivot). If condition fail, stop i there
                 3. Move j left: while(A[j] > Pivot). If condition fail, stop j there
                 4. If(i < j). Swap. But remember i & j indexes & also Pivot remains the same
                 5. If(i >= j). Swap with the Pivot element. Here very important, once the Pivot is sorted, the left
                    elements to it becomes a sub-array & also right elements become a sub array.
             
            * Example mental replay (no code)
           
              Given array
              [8, 3, 6, 1, 4, 5, 2]

              Step 1 Pick pivot

                     Pivot = 8 (first element)

                     Think:

                     “Everything smaller goes left, bigger goes right”

              Step 2️⃣ Partition around pivot

                     After partition:

                     [2, 3, 6, 1, 4, 5 | 8]


                     ✔ Pivot 8 is fixed forever

             Step 3️⃣ Recurse on left sub-array
                      [2, 3, 6, 1, 4, 5]


                      Pivot = 2

                      After partition:

                      [1 | 2 | 6, 3, 4, 5]


                       ✔ Pivot 2 fixed

            Step 4️⃣ Recurse on right sub-array
                      [6, 3, 4, 5]


                      Pivot = 6

                    After partition:

                     [5, 3, 4 | 6]


                    ✔ Pivot 6 fixed

            Step 5️⃣ Continue recursively
                     [5, 3, 4]


                     Pivot = 5

                     After partition:

                    [4, 3 | 5]


                    ✔ Pivot 5 fixed

           Step 6️⃣ Final small partition
                    [4, 3]


                   Pivot = 4

                  After partition:

                  [3 | 4]


                  ✔ Pivot 4 fixed

            ✅ Final sorted array
            [1, 2, 3, 4, 5, 6, 8]

       2. MERGE SORT (THE BEST)
            WHY?  - Guaranteed O(nlogn)
                  - Stable
                  - Backbone of external sorting and linked list sorting, Suitable for 'linked list'
                  - Drawback: Extra Space is needed (Not In Place sorting like 'Quick Sort')
                  - https://www.youtube.com/watch?v=ak-pz7tS5DE&list=PLDN4rrl48XKpZkf03iYFl-O29szjTrs_O&index=35
            
              * Key mental model (Algorithm): MergeSort → “Divide completely, then merge cleanly”
                1 Divide step (Recursion)

                  - Always divide the array into two halves using mid
                  - Keep dividing until each sub-array has only one element
                  - One element is already sorted — no work needed

                 📌 Important: MergeSort does nothing useful until the divide is complete

                2 Merge step (Where sorting actually happens)

                  - Take two sorted sub-arrays (left & right)
                  - Compare elements one by one
                  - Copy the smaller element first into the original array
                  - Continue until one sub-array is exhausted
                  - Copy the remaining elements as-is (already sorted)

                3 Index mental model (very important)

                  - Left and Right arrays start at index 0
                  - Original array writing starts at index low
                  - Merge only happens in the range low → high
                
            🧪 Example mental replay (no code)

                  For array:

                 [8, 3, 6, 1]


                 MergeSort thinks:

                  Split → [8,3] & [6,1]
                  Split → [8][3] & [6][1]
                  Merge → [3,8] & [1,6]
                  Merge → [1,3,6,8]

                  - In place
                  - Good when worst case matters 
            
            Just Watch this: https://www.youtube.com/watch?v=Q_eia3jC9Ts&t=4s

       4. INSERTION SORT (FOUNDATIONAL)
            WHY?  - Very fast for small or nearly sorted data
                  - Used internally by real algorithms (hybrid sorts) 
                  - https://www.youtube.com/watch?v=yCxV0kBpA6M&list=PLdo5W4Nhv31bbKJzrsKfMpo_grxuLl8LU&index=98
              * Key mental model (Algorithm): InsertionSort → “Insert element into sorted left part”
                
                1. The basic idea is to start with considering left side, first start with just 1 element and think of
                   this side as the 'sorted sublist'.
                2. The right side as the 'Unsorted sublist. Now consider the first element on the unsorted list to be the
                    the element to start comparing with the left sorted sublist.
                3. So what you have to check? Is the left sublist value greater than the element being compared. Than 
                   obviously you have to move the element to the left side before the existing left sublist element since
                   the right sublist element is lesser than the that.
                4. Keep this comparison logic, until you get the left sublist completely sorted. At the end, this will be the
                   sorted list.
                
                Algorithm:

                    for(int i = 1; i < n ; i++)
                    {
                      temp = a[i];
                      j = i - 1;

                      while(j >= 0 && a[j] > temp)
                      {
                        a[j+1] = a[j];
                        j--;
                      }
                      a[j+1] = temp;
                    }    

                    Time complexity: Best case =  O(n), Worst case = O(n2) 
                    Space complexity: O(1) [when space usage is just 'Constant']

            WHY?  - Just educational

              * Key mental model (Algorithm): BubbleSort → “Big values bubble to the end”

                 - You will use two for loops.
                 - The outer loop will keep the count of number of passes
                 - The inner loop will consists of adjacent comparisons until the 'largest' element bubbles to the end

              Algorithm:

                  for(int i = 0 ; i < n-1 ; i++)
                  {
                      for(int j = 0; j < n-1-i; j++)  [NOTE: you will have 'i' also here, why since with wach pass the last 'i' elements will already be sorted]

                          {
                              temp = a[j];

  how your 'optimizing' the solution based upon your own needs. This may not be true to everyone but you have figured outer
  this is the optimal solution for you. This is called 'Greedy Method'.
